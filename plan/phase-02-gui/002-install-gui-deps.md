# Verify gui/ installs cleanly with bun

## Purpose and scope

Confirm that `gui/` dependencies are correctly installed via the bun workspace (from the root install in Phase 1) and that the gui build succeeds with bun.

## Requirements

1. **Verify workspace install covered gui/**: Since `gui/` is listed in the root `"workspaces"`, `bun install` at the root in Phase 1 task 002 should have already populated `gui/node_modules/` or linked it from the root. Confirm this is the case.

2. **Delete `gui/package-lock.json`**: This file was generated by npm and is no longer valid. Delete it and confirm it is not regenerated.
   ```bash
   rm gui/package-lock.json
   ```

3. **Run `make build.gui`** from the repo root (or `make build` from `gui/`) to verify the build pipeline works end-to-end:
   ```bash
   make -C /Users/zane/playground/moduleforge/users-module build.gui
   ```

4. **Commit**: `chore(bun-migration): remove gui/package-lock.json`

## Validation

- `gui/package-lock.json` does not exist
- `make build.gui` exits with code 0
- `gui/dist/` is populated with build artifacts (index.js, index.mjs, index.d.ts, index.css)
- No npm or pnpm invocations appear in make build.gui output
